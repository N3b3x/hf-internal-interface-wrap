Detailed Description

No-op mutex backend – all operations compile to nothing.

This is the default backend when no RTOS or threading support is configured. Handle types are minimal (uint8_t) so mutex members occupy a single byte. Every operation is constexpr / inline and produces no machine code.

This struct also serves as the DOCUMENTATION of the backend contract: any concrete backend must provide the exact same set of static methods with compatible signatures.
