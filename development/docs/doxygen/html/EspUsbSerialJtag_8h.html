Detailed Description

ESP32-S3 / -C3 / -C6 USB Serial/JTAG console wrapper.

Concrete implementation of BaseUsbSerialJtag for Espressif chips that embed the USB Serial/JTAG controller (see ESP-IDF components/esp_driver_usb_serial_jtag/). Internally calls usb_serial_jtag_driver_install() once and forwards reads/writes to usb_serial_jtag_read_bytes() / usb_serial_jtag_write_bytes().

Why a wrapper?
Application code in main/ MUST NOT include driver/usb_serial_jtag.h directly — that would leak esp_err_t, TickType_t, and FreeRTOS into MCU-agnostic layers. By owning the driver behind BaseUsbSerialJtag, upper layers stay portable to alternate transports (host-side mock, future USB-CDC OTG implementation, etc.).
Coexistence with the IDF console
If CONFIG_ESP_CONSOLE_USB_SERIAL_JTAG=y, the IDF console driver is already installed before app_main runs and usb_serial_jtag_is_driver_installed() returns true. Initialize() detects this and treats the controller as already-good without calling usb_serial_jtag_driver_install() a second time (which would return ESP_ERR_INVALID_STATE). In that mode, the IDF printf / ESP_LOG infrastructure shares the controller with this wrapper; both paths push to the same TX ring without conflict.
Thread safety
The Espressif driver supports concurrent reader and writer tasks. This wrapper holds no internal state beyond initialized_ after Initialize(), so multiple writer tasks may call Write() concurrently. If you call Read() from more than one task, hold an external mutex.
Singleton helper
Most boards only ever need one instance (the controller is a single silicon block). Use EspUsbSerialJtag::Default() to grab a process-wide shared instance configured with the supplied config on first call.
